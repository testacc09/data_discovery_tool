# MCP.md — Спецификация Model Context Protocol

Этот документ описывает протокол взаимодействия между ИИ-агентами и инструментом поиска данных. Протокол позволяет агентам автономно исследовать метаданные источников данных компании.

---

## 1. Обзор возможностей

Сервер MCP предоставляет агенту набор инструментов (`tools`) для выполнения задач Data Discovery. Все операции работают поверх гибридного движка поиска, объединяющего векторное сходство (семантика) и точное совпадение (keywords).

---

## 2. Список доступных инструментов

### `listSources()`
Возвращает список всех подключенных коннекторов.
* **Аргументы:** Нет.
* **Результат:** Список идентификаторов источников.

### `indexSource(sourceId: string)`
Принудительно запускает переиндексацию метаданных для указанного источника.
* **Аргументы:** * `sourceId` (string): Идентификатор источника данных.
* **Результат:** Статус выполнения (успех/ошибка).

### `search(query: string)`
Основной инструмент поиска.
* **Аргументы:**
    * `query` (string): Поисковый запрос на естественном языке.
* **Результат:** Список найденных таблиц/колонок.

### `getSchema(sourceId: string, path: string)`
Получение детальной структуры конкретной таблицы.
* **Аргументы:**
    * `sourceId` (string): Идентификатор источника.
    * `path` (string): Полный путь к таблице (например, "main.orders").
* **Результат:** Схема данных (имена колонок, типы данных) в JSON-формате.

### Возможности сервера:

Сервер поддерживает следующие операции:

- обнаружение подключенных источников данных;
- семантический поиск таблиц и колонок;
- получение схемы выбранной таблицы;
- работу с несколькими типами источников данных через единый интерфейс.

### Ограничения:

Сервер предназначен только для работы с метаданными.

Он не:

- выполняет SQL-запросы;
- изменяет данные;
- создает таблицы;
- предоставляет полный дамп базы данных.

Все операции ограничиваются поиском объектов и получением их схем.

## 3. Пример диалога с агентом

**Запрос агента:** *"Найди таблицы, связанные с информацией о пользователях"*

**Вызов инструмента:**
```json
{
  "name": "search",
  "arguments": {
    "query": "user contact information"
  }
}
```
**Ответ сервера:**
```json
[
  {
    "path": "production.users.email",
    "score": 0.92,
    "metadata": {"type": "TEXT", "description": "Primary user email"}
  },
  {
    "path": "crm.profiles.phone_number",
    "score": 0.85,
    "metadata": {"type": "TEXT", "description": "Customer contact"}
  }
]
```

---

## 4. Требования к агентам

При использовании агентов рекомендуется подавать им следующий системный промпт для корректной работы с инструментом:

*"Ты имеешь доступ к инструменту Data Discovery. Перед выполнением SQL-запросов всегда используй инструмент search, чтобы найти правильные названия таблиц и колонок. Если поиск дал низкий скор (< 0.5), уточни запрос. Используй get_schema для проверки типов данных перед написанием сложных джойнов."*
